## 2.2 技术选型明细

| 层级 | 技术 | 版本 | 选择理由 |
|------|------|------|----------|
| **后端框架** | Django | 4.2 LTS | 成熟稳定、ORM 强大、Admin 内置、生态丰富 |
| **API 框架** | Django Ninja | ≥1.0 | 类型注解、自动 OpenAPI、高性能、类 FastAPI 体验 |
| **异步任务** | Celery | 5.x | Python 生态最成熟的分布式任务队列 |
| **消息代理** | Redis | 7.x | 同时做缓存和消息队列，减少依赖 |
| **数据库** | PostgreSQL | 15 LTS | 支持 JSONB、全文检索、ACID |
| **对象存储** | MinIO | latest | S3 兼容，可本地部署也可切换到阿里云 OSS |
| **前端框架** | Next.js | 14 (App Router) | SSR/SSG 兼顾、RSC、与 Django 解耦 |
| **UI 组件库** | Ant Design | 5.x | 开箱即用、中后台场景成熟 |
| **HTTP 客户端** | Axios | latest | 前端 API 请求封装 |
| **AI 图像** | Stable Diffusion A1111 API | - | 最广泛的开源图像生成方案 |
| **虚拟试衣** | IDM-VTON | - | 效果领先，部署文档完善 |
| **LLM** | OpenAI / Claude API | - | 按需切换，统一 AIService 抽象 |
| **容器化** | Docker Compose | latest | 单机编排，一人运维友好 |

## 2.5 模块间通信方式

| 场景 | 方式 | 说明 |
|------|------|------|
| 前后端交互 | **RESTful JSON API** | Django Ninja 提供，Next.js 通过 Axios 调用 |
| 实时任务进度 | **SSE** | Django 通过 StreamingHttpResponse 推送 |
| 异步任务处理 | **Celery + Redis** | AI 生成任务异步执行，避免阻塞 |
| 定时同步 | **Celery Beat** | ERP 数据定时同步，缓存刷新 |
| 跨容器通信 | **Docker 网络** | 通过服务名（service name）访问 |
| 外部 AI 模型 | **HTTP API** | 调用 SD WebUI / VTON 容器 |

## 2.6 安全架构

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  客户端   │────▶│  Nginx   │────▶│  Django  │
└──────────┘     └──────────┘     └──────────┘
     │                │                │
     │ HTTPS          │ TLS            │ JWT
     │ Bearer Token   │ 静态资源        │ Session
     │                │ 反向代理        │ CSRF Token
```

- **传输层**：全链路 HTTPS（Nginx 终结 TLS）
- **认证**：JWT（access token 30min + refresh token 7d）
- **授权**：Django Permission 框架 + 自定义角色检查装饰器
- **限流**：django-ratelimit 或 Ninja 中间件
- **CORS**：django-cors-headers 白名单模式
- **敏感数据**：数据库连接串、API Key 通过环境变量注入，不上传 Git
