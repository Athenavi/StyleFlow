# 07 - 工作流与业务流程设计

> **版本**：v1.0 | **更新**：2026-06-26

---

## 7.1 设计原则

- **轻量级**：不使用独立工作流引擎（如 Airflow、Camunda），基于 Django Model + Celery Canvas 实现
- **可配置**：流程模板通过 JSON 配置定义，支持添加/修改节点
- **可追溯**：每一步操作记录到 WorkflowNode 表，完整审计日志
- **单人友好**：默认自动推进，无需手动触发；审批节点才需人工介入

## 7.4 流程引擎实现

### 7.4.1 流程定义加载

```python
# apps/workflow/engine.py
import json
from pathlib import Path

class WorkflowEngine:
    """轻量级工作流引擎"""

    _definitions = {}

    @classmethod
    def load_definition(cls, workflow_type: str) -> dict:
        """加载流程定义（从 JSON 文件或数据库）"""
        if workflow_type not in cls._definitions:
            path = Path(__file__).parent / 'definitions' / f'{workflow_type}.json'
            with open(path) as f:
                cls._definitions[workflow_type] = json.load(f)
        return cls._definitions[workflow_type]

    @classmethod
    def create_instance(cls, workflow_type: str, object_id: int,
                         title: str, created_by, data_snapshot: dict = None):
        """创建流程实例"""
        definition = cls.load_definition(workflow_type)
        instance = WorkflowInstance.objects.create(
            workflow_type=workflow_type,
            object_id=object_id,
            title=title,
            current_node=definition['initial'],
            created_by=created_by,
            data_snapshot=data_snapshot or {},
        )
        # 记录起始节点
        instance.nodes.create(node_name=definition['initial'], action='start')
        return instance
```

### 7.4.2 节点推进

```python
class WorkflowEngine:
    # ... 接上

    @classmethod
    def proceed(cls, instance: WorkflowInstance, user, action: str = 'approve',
                 comment: str = ''):
        """推进流程到下一节点"""
        definition = cls.load_definition(instance.workflow_type)
        current_node_def = cls._find_node(definition, instance.current_node)

        if action == 'reject' and 'reject_to' in current_node_def:
            next_node = current_node_def['reject_to']
        elif action == 'approve':
            next_nodes = current_node_def.get('next', [])
            if not next_nodes:
                raise ValueError("当前节点已是终态")
            next_node = next_nodes[0]  # 默认走第一个分支
        else:
            raise ValueError(f"不支持的操作: {action}")

        # 记录节点操作
        instance.nodes.create(
            node_name=instance.current_node,
            handler=f"{user.get_full_name() or user.username}({user.id})",
            action=action,
            comment=comment,
        )

        # 更新实例状态
        next_node_def = cls._find_node(definition, next_node)
        is_terminal = not next_node_def.get('next')
        instance.current_node = next_node
        instance.status = 'completed' if is_terminal else 'running'
        instance.save()

        # 如果是终态，触发回调
        if is_terminal:
            cls._on_completed(instance)

        return instance

    @classmethod
    def _find_node(cls, definition, node_name):
        for node in definition['nodes']:
            if node['name'] == node_name:
                return node
        raise ValueError(f"节点 {node_name} 不存在")

    @classmethod
    def _on_completed(cls, instance):
        """流程完成回调"""
        if instance.workflow_type == 'style_development':
            # 款式开发完成 → 回写 ERP
            from apps.erp.tasks import sync_style_to_erp
            sync_style_to_erp.delay(instance.object_id)
```

## 7.5 业务流程详述

### 7.5.1 商品企划 → 设计

```
1. 设计师创建企划（手动输入或 AI 辅助生成）
2. 企划内容包括：波段、品类、风格关键词、参考图片
3. 系统自动创建 WorkflowInstance (type=style_development)
4. 进入 designing 节点
5. 设计师在 AI 设计工坊生成款式图
6. 选择满意款式 → "提交评审"
7. 流程进入 review 节点
```

### 7.5.2 设计评审

```
1. 管理员/主管看到待办事项
2. 查看设计稿详情、Prompt、AI 生成参数
3. 操作：
   - ✅ 通过 → 进入 techpack 节点
   - ❌ 驳回 → 回到 designing 节点，设计师修改
4. 驳回时可填写反馈意见
```

### 7.5.3 工艺单自动生成

```python
# apps/techpack/tasks.py
@shared_task
def generate_techpack(design_id: int):
    """AI 自动生成工艺单初稿"""
    design = Design.objects.get(id=design_id)

    # 1. 调用 LLM 提取设计参数
    llm = get_llm_service()
    schema = {
        "type": "object",
        "properties": {
            "collar_type": {"type": "string"},
            "sleeve_type": {"type": "string"},
            "length": {"type": "string"},
            "fabric_recommendation": {"type": "string"},
            "process_notes": {"type": "string"},
        }
    }
    result = llm.extract_json(
        prompt=f"根据以下设计描述提取工艺参数：\nPrompt: {design.prompt}\n图片URL: {design.image_url}",
        schema=schema,
    )

    # 2. 匹配工序模板
    processes = ProcessTemplate.objects.filter(
        category=design.category, is_active=True
    ).values('process_name', 'standard_time', 'unit_cost')

    # 3. 创建工艺单
    techpack = TechPack.objects.create(
        design=design,
        title=f"{design.title} 工艺单",
        fabric_description=result.get('fabric_recommendation', ''),
        process_steps=list(processes),
        status='draft',
    )
    return techpack.id
```

### 7.5.4 核工价流程

```
1. 工艺单生成后，流程自动进入 costing 节点
2. 系统自动匹配工序模板计算标准工价
3. 财务/管理员审核工价明细
4. 操作：
   - ✅ 通过 → 进入 approval 节点
   - ❌ 驳回 → 回到 techpack 节点修改工艺
```

## 7.6 待办与通知

### 待办事项生成

```python
# apps/workflow/tasks.py
def refresh_user_todos(user):
    """刷新用户的待办事项"""
    # 查询所有当前节点 handler_role 包含该用户角色的流程
    user_role = user.profile.role
    pending_instances = WorkflowInstance.objects.filter(
        status='running',
    )
    todos = []
    for inst in pending_instances:
        definition = WorkflowEngine.load_definition(inst.workflow_type)
        current = WorkflowEngine._find_node(definition, inst.current_node)
        if user_role in current.get('handler_role', []):
            todos.append({
                'instance_id': inst.id,
                'title': inst.title,
                'node_label': current['label'],
                'created_at': inst.created_at,
            })
    return todos
```

## 7.7 扩展性设计

| 场景 | 方案 |
|------|------|
| 新增流程类型 | 在 `definitions/` 下新建 JSON 文件，注册到 WorkflowInstance.WORKFLOW_TYPES |
| 修改流程节点 | 编辑 JSON 配置，已有运行中的实例不受影响（按原配置执行） |
| 新增分支条件 | 在 node 中增加 `conditions` 字段，`proceed()` 根据条件选择不同 next |
| 定时触发 | Celery Beat 定期检查，满足条件自动推进 |
| 并行节点 | 使用 Celery group 实现子任务并行 |
